# Отчет по работе с `rates.tsv`

## Постановка задачи

### Цель проекта

Необходимо разработать модель машинного обучения для определения наличия **личной истории** в отзыве пользователя.

На сайте ПроДокторов пользователи публикуют отзывы о врачах и медицинских учреждениях. Часть отзывов отклоняется модераторами по причине **«Нет личной истории»**, после чего автору предлагается дополнить текст.

Недостаток существующего процесса заключается в том, что между отправкой отзыва и получением ответа модератора может пройти значительное время. За этот период пользователь нередко теряет интерес к редактированию отзыва, что снижает вероятность его повторной публикации.

Целью проекта является создание модели, которая позволит автоматически определять отсутствие личной истории непосредственно в момент отправки отзыва и своевременно предлагать пользователю дополнить текст.

> При формировании датасета SQL-запросы частично генерировались с использованием GPT, поэтому в данных присутствуют различные аномалии, требующие дополнительной обработки.

---

## Описание набора данных

Датасет расположен в файле `data/rates.tsv`.

Используемые признаки:

| Признак | Описание |
|---------|----------|
| `text` | Текст отзыва |
| `is_lpu_rate` | Тип отзыва: `1` — отзыв о клинике, `0` — отзыв о враче |
| `accepted` | Целевая переменная. `1` — отзыв принят, `0` — отклонён по причине «Нет личной истории» |

Поле `text` имеет следующую структуру:

```text
[Общий комментарий]
[Понравилось: ...]
[Не понравилось: ...]
```

В целевой переменной наблюдается существенный дисбаланс: количество объектов класса `0` значительно превышает количество объектов класса `1`.

- полные дубликаты;
- частичные копии записей.

Все найденные дубликаты были удалены.

## Некорректные данные

В датасете присутствовали:

- повреждённые тексты;
- строки, содержащие только специальные символы;
- HTML/XML-разметка;
- отзывы на иностранных языках;
- прочие неинформативные записи.

После очистки датасета были сформированы дополнительные числовые признаки.

## Дополнительные признаки

| Признак | Описание |
|---------|----------|
| `mean_word_length` | Средняя длина слова |
| `word_count` | Количество слов |
| `mean_sent_length` | Средняя длина предложения |

Следует отметить, что признаки `word_count` и `mean_sent_length` обладают высокой корреляцией, что может привести к возникновению мультиколлинеарности. Для решения данной проблемы далее использовался алгоритм **Sequential Feature Selector**, позволяющий автоматически исключить избыточные признаки.

# Нейросетевые модели

Помимо классических алгоритмов машинного обучения были исследованы современные нейросетевые архитектуры для обработки естественного языка.

Цель экспериментов заключалась в оценке эффективности контекстных языковых моделей по сравнению с традиционным представлением текста с использованием TF-IDF.

В ходе работы были рассмотрены три подхода:

1. Полносвязная нейронная сеть с использованием обучаемых эмбеддингов.
2. Гибридная архитектура **ruBERT + LSTM**.
3. Полное дообучение модели **ruBERT** (Fine-Tuning).

# Использование ruBERT + LSTM

Следующим этапом стало применение предобученной языковой модели **ruBERT**.

В отличие от предыдущего подхода, вместо обучения собственных эмбеддингов использовались контекстные представления слов, полученные с помощью ruBERT.

Архитектура модели включала следующие компоненты:

1. Предобученная модель ruBERT.
2. Заморозка весов языковой модели.
3. Извлечение скрытых представлений предложений.
4. Двухслойная LSTM.
5. Полносвязный классификатор.

Использование LSTM позволило дополнительно учитывать последовательность слов и взаимосвязи между предложениями.

Схема модели:

```text

      ▼
Контекстные эмбеддинги
      │
      ▼
BiLSTM
      │
      ▼
Dropout
      │
      ▼

      │
      ▼
```

### Анализ обучения

Несмотря на ожидания, полное дообучение ruBERT не привело к улучшению качества классификации.

В процессе обучения наблюдались следующие особенности:

- снижение качества на валидационной выборке;
- увеличение разрыва между обучающей и тестовой ошибками;
- рост функции потерь после нескольких эпох.

Подобное поведение свидетельствует о начале переобучения модели.

Наиболее вероятными причинами являются:

- сравнительно небольшой объём обучающей выборки;
- большое количество параметров языковой модели;
- высокая вычислительная сложность полного Fine-Tuning.

### Вывод

Полное дообучение ruBERT не позволило улучшить качество классификации.

Наиболее эффективным оказался вариант с **замороженной языковой моделью** и дополнительным классификатором на основе LSTM.

Такой подход обеспечивает:

- более стабильное обучение;
- меньшее количество обучаемых параметров;
- снижение риска переобучения;
- лучшее качество классификации на тестовой выборке.

Таким образом, использование предварительно обученной модели **ruBERT** в качестве экстрактора признаков оказалось более эффективным решением для рассматриваемой задачи, чем её полное дообучение.

## Анализ результатов

Проведённые эксперименты позволяют сделать несколько важных выводов.

Классические алгоритмы машинного обучения демонстрируют высокое качество классификации при использовании TF-IDF представления текста. Наиболее успешной моделью данной группы стал **CatBoost**, показавший лучшие значения Accuracy и Macro F1 среди всех исследованных классических алгоритмов.

Использование дополнительных статистических признаков позволило получить лишь незначительный прирост качества. Основная часть полезной информации содержится непосредственно в текстовом представлении отзывов.

Применение методов балансировки классов улучшило качество распознавания меньшего класса. Наиболее эффективным оказался метод **Random Under Sampling**, несмотря на уменьшение объёма обучающих данных.

Использование нейронных сетей позволило перейти от анализа отдельных слов к обработке контекстной информации.

Модель с обучаемыми эмбеддингами показала достойный результат, однако уступила архитектурам, использующим предварительно обученные языковые модели.

Лучшее качество классификации было достигнуто при использовании гибридной архитектуры **ruBERT + LSTM**, которая сочетает контекстные представления текста с возможностями рекуррентных сетей по моделированию последовательностей.

Полное дообучение ruBERT не привело к улучшению результатов. Наоборот, увеличение количества обучаемых параметров привело к появлению признаков переобучения и снижению качества на тестовой выборке.

В процессе анализа данных были выявлены и устранены дубликаты, повреждённые записи, HTML-разметка, отзывы на иностранных языках и другие источники шума, способные негативно повлиять на качество обучения моделей.

В качестве базового способа представления текста использовалась TF-IDF-векторизация. Среди классических алгоритмов наилучшие результаты показала модель **CatBoost**, продемонстрировавшая высокую устойчивость и хорошее качество классификации.

Дальнейшее развитие проекта было связано с использованием глубокого обучения. Исследование показало, что применение контекстных языковых моделей позволяет учитывать семантику текста значительно эффективнее по сравнению с традиционными методами векторизации.

Максимальное качество было достигнуто гибридной моделью **ruBERT + LSTM**, использующей предварительно обученные контекстные эмбеддинги без полного дообучения языковой модели. Данный подход оказался наиболее устойчивым и обеспечил лучшие значения основных метрик качества.

Эксперимент с Fine-Tuning ruBERT показал, что увеличение сложности модели не всегда приводит к росту качества. При существующем объёме обучающих данных полное дообучение языковой модели сопровождалось переобучением и ухудшением результатов на тестовой выборке.

Таким образом, поставленная цель исследования была достигнута. Разработанная модель способна автоматически определять отсутствие личной истории в отзыве и может использоваться в качестве интеллектуального помощника при публикации отзывов на медицинском портале.

# Возможные направления развития проекта

Для дальнейшего повышения качества классификации могут быть рассмотрены следующие направления:

- увеличение объёма размеченного датасета;
- применение современных языковых моделей (RuModernBERT, XLM-RoBERTa, mDeBERTa);
- использование ансамблей нескольких моделей;
- оптимизация гиперпараметров с использованием Optuna;
- применение методов аугментации текстовых данных;
- исследование более эффективных методов балансировки классов.

Реализация перечисленных направлений позволит дополнительно повысить качество классификации и устойчивость модели на новых данных.
